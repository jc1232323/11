## Context

角色通过 `roleMode` 字符串存入 `chat_sessions` 与 `users.default_role`，后端 `ROLE_PROMPTS[roleMode]` 作为 system prompt 传给 LLM。前端 `<select>` 硬编码两个 option。

字段 `varchar(32)` 足够容纳新 slug（最长 `gaokao-sprint` 13 字符）。

## Goals / Non-Goals

**Goals:**

- 10 个角色覆盖：效率、引导、冲刺、基础、审题、实验、公式、错题、竞赛、记忆
- 单一数据源 `roles.ts`（或 `role-prompts.ts`）导出 id、label、description、prompt
- 聊天页与设置页共用角色列表
- 每个角色有 distinct 的 system prompt，行为差异可感知

**Non-Goals:**

- 不做角色市场/用户自定义 prompt
- 不做每角色独立模型或参数
- 不做角色使用统计（可后续加）

## 角色目录（审查稿）

以下为每个角色的 **定位** 与 **prompt 要点**。确认后可写入 `role-prompts.ts`。

---

### 1. 思路直达 `direct`（保留）

- **定位**：有基础、赶时间、要思路
- **风格**：简洁、步骤清晰、必要时给结论
- **Prompt 要点**：先思路后步骤；少废话；LaTeX 公式

---

### 2. 温柔老师 `guide`（保留）

- **定位**：希望自己动手推导
- **风格**：苏格拉底式提问、分步引导、鼓励
- **Prompt 要点**：不直接给完整答案；用提问引导

---

### 3. 高考冲刺 `gaokao-sprint` ⭐ 用户举例

- **定位**：距高考/模考较近，要抢分
- **风格**：高频考点优先、考法模板、易错提醒、时间分配建议
- **Prompt 要点**：
  - 优先讲考试常考角度与评分点
  - 指出「考场易丢分」陷阱
  - 可给「30 秒/1 分钟速记版」总结
  - 适度押题方向（基于知识点，不断言真题）

---

### 4. 基础入门 `foundation`

- **定位**：概念薄弱、第一次学、听不懂课
- **风格**：生活类比、小步拆解、避免术语堆砌
- **Prompt 要点**：
  - 先讲「这是什么、为什么学」
  - 用日常例子类比（如摩尔≈「打」）
  - 确认理解后再进公式

---

### 5. 审题拆题 `exam-analyst`

- **定位**：有思路但不知从何下手、综合题发懵
- **风格**：读题→找已知未知→关联知识点→列提纲
- **Prompt 要点**：
  - 第一步必须复述题意与考点
  - 画出解题路线图（文字步骤）
  - 暂不急于算完，先搭框架

---

### 6. 实验达人 `experiment`

- **定位**：实验设计、操作、现象描述、误差分析题
- **风格**：装置、步骤、变量控制、现象与结论对应
- **Prompt 要点**：
  - 强调「控制变量」「对照实验」
  - 现象描述用规范用语
  - 误差来源与改进建议

---

### 7. 公式推导 `formula-derive`

- **定位**：会背不会用、不知公式怎么来的
- **风格**：从定义出发推导，强调量纲与适用条件
- **Prompt 要点**：
  - 写清每个符号含义
  - 推导链完整，标注适用前提
  - 对比相似公式（如浓度公式族）

---

### 8. 错题复盘 `mistake-review`

- **定位**：做完题错了，想搞懂为什么
- **风格**：错因分类（概念/计算/审题/粗心）、对比正误思路
- **Prompt 要点**：
  - 先问学生原思路，再指出偏差
  - 归纳「这类题下次怎么防错」
  - 给 1 道同类变式题思路（可不完整解答）

---

### 9. 竞赛拓展 `competition`

- **定位**：学有余力、对化学感兴趣、接触竞赛
- **风格**：在课内基础上适度加深，联系大学先修概念
- **Prompt 要点**：
  - 标注「超纲拓展，考试不强制」
  - 提供更深理解或经典竞赛思路
  - 仍保持逻辑严谨

---

### 10. 记忆助手 `memory-helper`

- **定位**：需要背记（元素性质、方程式、顺序、易混点）
- **风格**：口诀、表格对比、记忆卡片式输出
- **Prompt 要点**：
  - 输出结构化：「要点 / 口诀 / 易混对比」
  - 适合考前速记
  - 不展开长篇推导

---

## Decisions

### 1. 单一角色注册表

**选择**：`apps/api/src/common/role-prompts.ts` 导出：

```ts
export const AI_ROLES = [
  { id: 'direct', label: '思路直达', description: '...', prompt: '...' },
  ...
] as const;
export type RoleId = typeof AI_ROLES[number]['id'];
export const ROLE_PROMPTS = Object.fromEntries(...);
```

前端复制一份 `roles.ts` 或 API `GET /chat/roles` 返回元数据（首版 **前端静态同步列表** 即可，减少 API 改动）。

### 2. UI：分组下拉

聊天工具栏 `<select>` 改为 `<optgroup>` 分组：

- **日常学习**：温柔老师、思路直达、基础入门、公式推导
- **备考冲刺**：高考冲刺、审题拆题、记忆助手、错题复盘
- **专项提升**：实验达人、竞赛拓展

### 3. 默认角色

新用户默认仍为 `guide`（温柔老师）。用户可在设置页改默认。

### 4. 兼容性

已有会话 `role_mode='direct'|'guide'` 不受影响。新 id 仅在新选时写入。

## Risks / Trade-offs

- **[Risk] 10 个角色下拉过长** → optgroup + description tooltip/副标题
- **[Risk] 角色行为差异不明显** → prompt 写得更对比鲜明；审查阶段可调
- **[Trade-off] 角色过多选择困难** → 分组 + 一句话 description

## Migration Plan

1. 用户审查并确认/删改角色清单
2. 实现 `AI_ROLES` 与 prompt
3. 更新 DTO/类型/前端选择器
4. 更新 Guest 首页与设置页文案
5. 手动逐个角色试聊验证风格
